# UI Generation Prompt for Google Stitch

Use the following prompt with Google Stitch (or similar AI UI generators) to generate the Next.js frontend for the Zomato AI Recommender.

---

## Prompt: Premium Next.js Restaurant Recommendation Interface

**Objective:** Create a high-end, modern, and highly interactive restaurant discovery interface using **Next.js**, **Tailwind CSS**, and **Framer Motion**. The design should be "Zomato-inspired" but elevated with a "Premium AI Expert" aesthetic.

### 1. Visual Style & Theme
- **Color Palette:** Deep Charcoal/Midnight background (#0B0C10), Electric Purple/Neon Pink accents for AI features, Gold for ratings, and soft Emerald for "Healthy/Open" statuses.
- **Aesthetics:** Glassmorphism (frosted glass effects), subtle mesh gradients in the background, and 1px border highlights.
- **Typography:** Modern sans-serif (Inter or Outfit). Use large, bold headings for the AI summary and clean, medium weights for restaurant details.

### 2. Layout Structure
- **Global Header:** Minimalist nav with a "System Health" pulse indicator (green glow).
- **Hero / Input Section:** 
    - A sleek sidebar or a centered "Discovery Hub" where users input preferences.
    - Form fields: Location (Searchable dropdown), Cuisine (Multi-select pills), Budget (Segmented control: $, $$, $$$), Min Rating (Star slider), and "Special Requests" (Textarea for family-friendly, date night, etc.).
- **Main Content Area:**
    - **AI Summary Blurb:** A featured glowing card at the top containing a punchy, 2-sentence hype summary generated by the AI expert.
    - **Results Grid:** A responsive 2-column grid of "Restaurant Cards".
- **Empty State:** A beautiful illustration or "AI thinking" animation when no filters are applied.

### 3. Component Details
- **Restaurant Card:**
    - High-contrast name and rating badge.
    - Tags/Pills for cuisines and cost tiers.
    - **"AI Perspective" Section:** A dedicated, slightly indented block with a unique background tint (light purple) containing the AI's explanation of why this restaurant was picked.
- **Loading State:** Implement sophisticated skeleton loaders (shimmer effect) that mimic the card layout.
- **Animations:** Use Framer Motion for stagger-load entry animations on cards and smooth transitions between search states.

### 4. Technical Constraints (Next.js)
- Use **App Router** structure.
- Implement **Server Components** for data fetching where possible, but keep the Form and Cards as **Client Components** for interactivity.
- Ensure the UI is **fully responsive** (Mobile-first, scaling to wide desktop).
- Use **Lucide-React** icons for all iconography.

---

### How to use this prompt:
1. Copy the text above starting from **"Objective"**.
2. Paste it into Google Stitch or your preferred AI UI generation tool.
3. Specify that you want the output as a **Next.js 14+ project using Tailwind CSS**.
